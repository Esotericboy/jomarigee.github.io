    Machine Information

    DC-4 is another vulnerable machine in the DC-series. However, unlike previous DC boxes there is only one flag on this machine. Thankfully there are multiple entry points.

    You can access this vulnerable machine here

    Information Gathering

    The target IP address isn't provided upon machine start, so I used the tool netdiscover to find the IP of the machine.

    ➜ sudo netdiscover -r 192.168.0.1/24

    Scanning/Enumeration

    Running a normal nmap scan on the target I see that port 22 and port 80 are open, which is nice because the box seems to be pretty normal so far.

    ➜ nmap -A -p- 192.168.0.151 -o nmap.txt

    Next, I ran a Gobuster scan, which actually didn't display anything interesting except for a command.php directory, that I couldn't visit yet.

    ➜ gobuster dir -u 192.168.0.151 -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -n -e -o dir.txt -x tml,php -t 30

    Login Brute Force

    I tried using Nikto and Nmap vulns scan to find any low hanging fruit, but nothing was found so I went to browse the main domain and found a login portal. This portal is different then others I've come across because you don't get an error message after entering the wrong credentials, weird.

    After smashing my head into the wall I was able to login successfully via the usage of Hydra, however, this was a nice tricky login portal at first. In my Hydra command I used :S=command at the end because command.php was the only place I couldn't access. This thinking was also derived because I didn't know what an error message looked like, since none were give.

    ➜ hydra -l admin -P /home/kali/rockyou.txt 192.168.0.151 http-post-form "/login.php:username=^USER^&password=^PASS^:S=command" -I

    Reverse Shell

    Once logged in I'm only given the option to select command and go to the command.php directory, go figure that's why the command success message worked.

    After clicking the command Button I'm given the option to click one of three buttons to run commands. I clicked the List Files option and as expected the files were listed. However, these were the same directories found with my Gobuster scan. This time they were displayed in the Linux Filesystem. Along with the command I ran, interesting.

    I capture that request via Burp Suite to see if there was any filtering in place and to my surprise there wasn't. Each command entered is sent unfiltered which opens up easy reverse shell possibilities.

    To get the reverse shell I used Pentest Monkey's reverse shell cheat sheet, with the netcat command. I sent the command via Burp Suite repeater like the image below editing the radio element.

    Now I just make a proper TTY Shell with so that I can handle all the possible commands I may need to use.

    Brute Force SSH

    When looking around the file system you see multiple users, but the only user that matters is Jim, because their files contain a long list of old passwords. These can potentially be used to brute force ssh.

    I made a simple python web server, so that I could download those passwords onto my local machine and begin Brute Forcing the ssh service.

    I use Hydra along with the password file from Jim to brute force his SSH login and get his password!

    ➜ hydra -l jim -P old-passwords.bak 192.168.0.151 -t 4 ssh -I

    Fun Mail

    After looking around Jim's file system I couldn't find anything of value at all, so I decided to import LinPEAS for further enumeration. There was nothing of interest that came from the scan.

    However, at the end of the scan I was told that I have mail in /var/mail/jim , which was pretty odd. This message wasn't a result of a scan, but the scan helped to draw attention to the mail filesystem.

    Which is where the password for another user, Charles, was located in an email to Jim!

    Privilege Escalation

    Now that I'm logged in as the user Charles I can go for a privilege escalation because as user Jim I wasn't in the sudeors file so that wasn't an option.

    In order to become root I don't even need a password, which is awesome! However cool that is I didn't know what teehee was, so after some googling I figured out the box's creator was making a joke with the name teehee because it's a play on the command tee.

    Root Flag

    After a few checks with a couple of fake commands I understand that the teehee command functioned similar to the tee command, but the tee command was restricted. So in order to become root I would need to use teehee, duh.

    I entered the same teehee command as above, this time I added the user I am (charles) to the sudoers path. Therefore allowing me to sudo su as the root user!!

    From here it was just a simple navigation to the root directory, then catting out that file to complete the box.

    Finished

    This box was a lot of fun to me because I had dealt with text filtering in boxes before, but not as unsuspecting as with this machine which was fun. Then the actual way to get root was fun with the mail prompt then the teehee command. ☺
