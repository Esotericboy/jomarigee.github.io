    Machine Information

    This machine like the last two in the DC series only requires you to get the root flag to complete this machine. DC-6 like DC-2 does need you to edit your host file, so that you can actually begin attacking the machine.

    You can find this Machine Here

    To add this machine to your host enter 

    ➜ subl /etc/hosts
    , you don't have to use Sublime, you can use any text editor you want. Once in your preferred text editor add the IP of the machine and 
    ➜ wordy
    next to it, then you're all set up.

    Along with that this message or clue however you take it should be taken because I didn't. That definitely cost me some time when trying to break this box, time that I could've saved, so go ahead and develop that password file lol.

    Scanning/Enumeration

    Doing a basic Nmap scan, I get back just ports 22 and 80 being open. Nothing interesting yet about these ports, but I know that on 80 the WordPress site is sitting which is nice.

    ➜ nmap -A 192.168.1.188 -o nmap.txt

    Running a Nmap vulnerable script scan is where things start to heat up. I'm told about the usual CSRF vulnerabilities as well as the location of the admin folder, which isn't a secret since this is WordPress. The cool part is I find potential service versions to exploit if need be along with the potential for an SQL injection. The best part is I now have a list of 5 different usernames for the application.

    ➜ nmap --script vuln 192.168.1.188 -o nmapvuln.txt

    My Gobuster scan didn't throw back anything more than I expected from a WordPress application, but its nice to have this as a reference though.

    ➜ gobuster dir -u 192.168.1.188 -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -n -e -o dir.txt -x html,php -t 30

    Exploit WP Login

    Visiting the main domain there is nothing interesting on the home page or the contact page, but on the about us page something small stands out. They talk about their main developer jens, who was a username that popped back from the nmap vulnerable scan. This username may prove to be valuable later.

    That was actually a rabbit hole that I fell down, trying to break in with just the username jens. That's a lesson for later is don't fall down the rabbit hole lol.

    To get out of that rabbit hole I created a file for the usernames called users.txt and then used that with the password list from earlier to get login creds for the WP application via WPScan. After letting the scan run for a good time, I get back a username and password!

    wpscan --url http://wordy/wp-login.php -t 40 -U users.txt -P passwords.txt

    Reverse Shell

    Now that I'm logged into the application its time to find either a reverse shell or the path I need to go down for getting a reverse shell. Looking around I don't see things I'm used to seeing with a WordPress Application, such as themes. The only thing that stands out is the activity monitor plugin, which makes sense because their security plugins are what they bragged about on the home page.

    I decided to pass that to searchsploit to see if anything was valuable with this system that I could exploit and there was, which is awesome!

    Getting SSH Creds

    Now that I'm in the system, I go to the home directory to see what users are there. I see the usernames that I had expected to be logged into the system, so time to check out each directory.

    In Marks directory he has a text file called things-to-do.txt which has a password for the user graham who has already been set up in plaintext. This is perfect because I can use it to login to SSH.

    The credentials found in mark's directory worked and I was able to login as the user graham's ssh session!

    File Hijacking

    Trying to immediately go for a privilege escalation I get told that to become root I don't need to enter a password and can run the backups.sh file in jens directory, so time to check that file out.

    Looking at the file it is just a basic script to start the webserver, nothing that I can leverage. I think that this is a script that needs to modified so that I can become jens.

    Checking out the files permissions and the group that I'm in (devs) I can modify this script!

    I modified the script by taking out the tar call command and added in a call to bash.

    Then I used the sudo perms to login as jens by running the modified backups.sh file.

    Now that I'm jens I see that to become root I don't need a password and can exploit the misconfiguration with nmap, so time to turn to GTFOBins.

    Thanks to GTFOBins I'm able to become root on this box! Next, all I do is navigate to the root folder and read the flag.

    Finished

    This was a fun machine with an easy entry point if you paid attention, but I didn't, so it took more time than I'd like to mention. All in all another great DC series VH box as usual. ☺
