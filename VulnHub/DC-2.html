    Machine Information

    DC-2 is just like the DC machine before it, coming in with a whopping five flags. However the most important one being the fifth and final one, root.

    You can find this Machine Here

    Information Gathering

    The target IP address isn't provided upon machine start, so I used the tool netdiscover to find the IP of the machine.

    ➜ sudo netdiscover -r 192.168.0.1/24

    Scanning/Enumeration

    Running an nmap scan I got back only port 80 being open right now, which is weird that there isn't an SSH port along with it. Nmap tells me that the website redirects you from the IP to http://dc-2/.

    ➜ nmap -A 192.168.0.131

    The Gobuster scan tells me that I'm dealing with a WordPress site, which is always nice to know already. Along with that I get some other directories to search through later, but first I need to setup the machine for proper usage.

    ➜ gobuster dir -u http://192.168.0.131 -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -n -e -x php,html,txt -t 30

    Setup

    Following the instructions from Vulnhub I set the machine up by going into my hosts by entering sudo vi /etc/hosts. From there I added the IP and dc-2 so that the WordPress site could work correctly.

    Flag One

    When visiting the main domain you can see on the front page the first flag along with a pretty nice plant as well.

    This first flag talks about how a normal wordlist (like rockyou) wouldn't work in order to login. However it also gives a hint to log in as another user if possible. The biggest hint the flag gives is the usage of this custom word list generator called CeWL. If you're using Kali Linux you can install this tool with a simple sudo apt-get install cewl.

    Then using the tool CeWL I got a wordlist that included letters as well as numbers, because I didn't want to leave anything to chance.

    ➜ cewl MACHINE_IP/index.php -w wordlist --with-numbers -d 3

    WPScan

    Since I'm dealing with a WordPress application the best tool to use for Brute Forcing the login page is WPScan. So using my CeWL list and the command below I got two login credentials.

    ➜ wpscan --url http://dc-2 -t 50 -P list.txt -e

    Flag Two

    In order to find the second flag I had to be logged in as Jerry and navigate to the Pages section and there was flag two just sitting there. Now flag one makes more sense with its hint because the user Jerry had more privileges then Tom.

    Flag two wasn't as much of a hint as flag one had been for me, but it does tip off that there are two ways to get the third flag.

    I realized that I didn't even need to login to the WordPress site to get the second Flag because its just hidden from the nav bar. Viewing the source code for the main page revealed that to me.

    Flag Three

    After looking around for a while I didn't have anything that I could delve deeper into so I figured it was time for more enumeration. So I started off with another nmap scan and got back an SSH port this time. Which is why I should've scanned for all the ports upfront.

    Using Tom's credentials found earlier I was able to login to his SSH session. Unfortunately I was unable to login to Jerry's session even with the credentials that worked for his WordPress login.

    The third flag was in Tom's directory sitting there like the other flags. Funny enough the session doesn't let you cat out files because its a restricted bash shell, so instead I had to use the less command to see it. The hint on the third flag tells me that I'll be logging in as either the user Jerry or another user to get the fourth flag.

    Flag Four

    First I needed to escape this restricted bash shell if I was going to do anything, so thanks to this article I was able to. I first found out that the text editor VIM was usable if you typed vi instead of vim . This was found because if you ls the contents of the usr bin file you can see what commands tom can run

    ➜ ls /usr/bin

    Then I just typed :set shell=/bin/sh pressed enter then deleted it. Next I entered :shell and after that I was greeted with a wonderful normal bash shell.

    I knew it was a bash shell because I could use commands like cd unlike previously.

    Lastly I set it to a normal shell so that I can actually enter commands and finish this box up. 

    ➜ export PATH=$PATH:/bin:/usr/bin

    Root Flag

    Now that I have an actual real proper bash shell I su as the user Jerry because you can't do sudo -l as the user Tom. To my wonderful delight you don't need a password to become root!!

    Using the wonderful resource that is GTFO Bins I'm able to become root by exploiting the misconfiguration in the usr/bin/git folder.

    Now I simply cat out the root flag, and the box is finished!!

    Finished

    I liked this machine because the flags gave great hints and this was the first box I'd dealt with that had a restricted bash shell. Overall a great beginner box for people like me.☺
