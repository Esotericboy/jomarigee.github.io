    Machine Information

    TomGhost is a CTF-style machine that covers the service tomcat in which you must gather two flags, the user and root text flag as usual.

    You can Join this TryHackMe Machine Here

    Scanning/Enumeration

    Running a regular Nmap scan on the target I see that port 22 is open along with DNS being port 53, strange. Looking down the list I see port 8009 and port 8080 is where the website for Apache Tomcat is located. From the Nmap scan, I also get a service version for Tomcat being 9.0.30.

    ➜ nmap -A 10.10.170.119 -o nmap.txt

    Throwing up a vulnerable Nmap scan I see that it tells me about the ever so common Sloworis CVE, which I don't care about because it deals with DDOS. The only interesting feedback I get from this scan is that /examples/ and /docs/ are potentially interesting folders.

    ➜ nmap --script vuln 10.10.170.119 -o nmapvuln.txt

    SSH Login

    Visiting http://10.10.119.68:8080/manager/ it told about a potential username and password that I could leverage, however I'm not sure where just yet. I tried poking around for a login portal of some sorts, but couldn't find one.

    I hit a brick wall and didn't know where to turn, so I went back to my nmap scan to get information to throw at searchsploit. I got back a valid hit for the service AJP(Jserv), which is often used alongside Tomcat. This can help me to leverage some credentials and get moving with this box. Looking back on this box, of course this exploit would be the one to work because the machine is called tomghost and the exploit is Ghostcat.

    Trying to get the exploit to work I am greeted with a message telling me I need to specify the host, port, and a file before I can actually follow through with this attack.

    However, I found out that I that I just need to specify the host that I'm attacking and the exploit will work perfectly fine. It gave me a pretty funny username to use, which is awesome. Now its time to take these credentials and login to the only place I know to, SSH.

    ➜ python 48143.py 10.10.119.68

    Using those credentials I was able to successfully login to SSH, time to keep rolling!!

    User Flag

    At first I thought a simple ls command would display the user flag, but unfortunately it didn't. I was instead meet by a pgp and asc file, so that means that I will have to decrypt this if I want to get root probably.

    The user flag was sitting in a different users directory. Once I bounced back and got into the user Merlin's directory I was able to find the user flag, which is nice.

    PGP Madness

    First thing I did was get the files on my home system, so I started a Python web server and downloaded them to begin working on them.

    After doing a little bit of googling I heard about pgp2john which is what I used to crack the PGP file and get a password back from the tryhackhackme.asc file.

    Now that I have the password that I need, I want to try and decrypt the credential.pgp file, so that this box can near the end. I enter the command below to try and import the ASCII armor as a key

    ➜ gpg --import tryhackme.asc

    With the armor file imported I press forward with decrypting the pgp file. Which works!! The output is a password for the user merlin, who is another user on the ssh service.

    ➜ gpg --decrypt credential.pgp

    Privilege Escalation

    Now with the credentials I just got from the PGP file I login as merlin on ssh, and it works! In merlin's directory the user flag sits, but I was already able to get that logged in as the other user, so this isn't a surprise.

    With the previous user I was unable to use sudo -l to escalate my privileges, but with merlin I can. After entering the command I'm told that this user doesn't need a password to become root as long as I abuse the misconfiguration in zip

    I went to the good ol resource of GTFObins and found the way to become root on this machine. Following the steps on the site I gained root access.

    Root Flag

    Now that I'm root on this machine, all that is needed is a simple navigation to the root folder and catting out the flag.

    Finished

    This was an awesome box because it was my first time dealing with PGP in a CTF style, let alone at all. It's really nice to see new concepts roll into CTFs.☺
