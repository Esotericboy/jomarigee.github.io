    Scanning/Enumeration

    Looking at the basic Nmap scan I see that port 21 is open and allowing Anonymous login. Along with that the standard ports 22 and 80 are open with a default Ubuntu page on port 80 

    ➜ nmap -A 10.10.0.0

    Using Gobuster for directory enumeration I just get back two subdirectories right now, with the most interesting being hackathons. I also now know that I'm dealing with a WordPress application. 

    ➜ gobuster dir -u http://10.10.0.0 -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -n -e -x php,html,txt -t 30

    /Hackathons

    There turned out to be nothing in the FTP server from the looking around that I did. I couldn't even upload anything to it, so I decided to move onto the Hackathon subdirectory. Which upon first glance doesn't provide much of anything.

    However, when inspecting the element I do see at the bottom a comment that says keep going and a weird string of text, but I'm not sure what its supposed to be used for yet, if anything.

    WordPress

    Going to the WordPress application so far nothing sticks out and it looks pretty normal as of now. The main domain for this box was just a default Apache home page, with nothing more hidden in the comments, so time to crack WPS.

    When on the login portal at /wordpress/wp-login.php I entered admin : admin to see if that was a possible login that'd work, but it didn't.

    However, when using elyana as the username (because they're an author) I got back a different error message, which means I can try and brute force this account for login credentials.

    Unfortunately, that was a rabbit hole because I missed a crucial step, the password was already given to me a while ago with a hint on the /hackathons page. The hint being Vinegar or Vigenere is the cipher used on Dvc W@iyur@123. Using this site with the key KeepGoing I got the passwordH@ckme@123.

    Reverse Shell

    After trying to exploit one of the plugins to no avail, I decided to use the old way of getting a reverse shell on a WPS application. First I navigated to http://10.10.86.250/wordpress/wp-admin/theme-editor.php?file=functions.php&theme=twentytwenty here and changed the php code to a reverse shell. Then to execute it I navigated to its location at http://10.10.86.250/wordpress/wp-content/themes/twentytwenty/functions.php and the shell was popped.

    While in Elyana's directory I found the user flag, but I couldn't access it because I kept getting permission denied. I tired other command's than cat, but they came of no avail. The hint.txt file did tell me that I would have to search around for Elyana's SSH password, so time to get dirty.

    There is more than one way to find the secret file with Elyana's password. The first one I used was putting linpeas on the system to check for any interesting or out of the ordinary files, which lead me to the private.txt file. Or the command find / -user elyana -type f 2>/dev/null can be used, which finds all mentions of the user Elyana in this system.

    User Flag

    Now with Elyana's recently found password I can login to their SSH account and get the user flag knocked out! In order to get the flag, you have to decode this because its in base64, so I just used this decoder since its easier than being on the terminal.

    Root Flag

    Going with the classic sudo -l I see that there is no password required to become root, so that's awesome. I turn to GTFO bins to figure out what command I need to run, so that I can be root. Then shortly after that, I'm able to get the root flag! The same thing with the user flag, in order to get the actual flag, you have to decode the base64 encoding and then the room is completed!!

    Finished

    This was honestly a great box and I had fun doing it with just one exploit. Now I'm going to go back through and see how many other exploits are possible on this machine! I really liked the base64 encoding touch at the end because it threw me off at first lol. ☺
