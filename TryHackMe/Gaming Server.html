    Scanning/Enumeration

    First thing is I do my go to nmap scan to see whats on this machine. I get back the classic port 22 for ssh and port 80 for an http site, so nothing strange yet.

    ➜ nmap -A 10.10.13.48

    Looking at the scan we can see the following ports and services are open:

    • ● Port 22/tcp - SSH - (OpenSSH 7.6p1)
    • ● Port 22/tcp - HTTP - (httpd 2.4.29)

    Next thing I did was run a gobuster scan on the box which threw back a lot of sub directories to investigate. I figured it would be easiest to start with the main domain and work down from there. 

    ➜ gobuster dir -u http://10.10.13.48 -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -n -e -x php,html,txt -t 30

    Main Domain

    Browsing the main domain it is pretty big with what at first appears to have a lot of functionality compared to most TryHackMe websites. I decided to poke around this site a little bit more.

    While clicking around I found a button called uploads which was also found with the gobuster scan.

    Clicking on this button I'm taken to a directory listings page, so now I'm starting to think this is where the reverse shell will come in for SSH capabilities.

    Going back to the results from my gobuster scan I look back over the results because its been running in the background. Its popped back a New directory called /secret. Which is definitely interesting. However, my attention is first drawn to /about.php because there is already an about directory, so why where there need to be one with a php extension.

    Browsing the weird page I see that this is in fact weird for a reason. Its the place that I can upload a shell and get it to trigger too!

    Before I go down that rabbit hole I decided to check out the other two gobuster scan results. First I went to the robots.txt page, but it was just another way to navigate to the uploads directory. So now onto the "secret" directory.

    John & SSH

    In the secret directory I find a file that says secret key, which are most likely for the SSH keys on this box.

    Looking at the file, it definitely is keys for SSH. I note the potential reverse shell possibilities I had seen before, but move onto trying to use these keys because it feels right.

    I download the key onto my machine, so that I can use the tool John The Ripper on it.

    ➜ wget http://10.10.13.48/secret/secretKey

    I change the permissions on the file so that I can use it to try and login with, but it asks me for a password. So that's where the password list I found earlier comes into play.

    So now comes the time to use John The Ripper to crack the hash, so I can get the correct passphrase for the SSH login. 

    ➜ python /usr/share/john/ssh2john.py secretKey > sec.hash sudo john --wordlist=dict.lst sec.hash 
    ➜ sudo john --wordlist=dic.lst sec.hash
