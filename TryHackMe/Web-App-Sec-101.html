    Question 1

    What version of Apache is being used?

    Answer: 2.4.7

    To answer this question you can do an Nmap scan like the picture below with the command being nmap 10.10.201.59 -A -o nmap.txt, or you can use a Firefox add-on called Wappalyzer which will provide information about the website's design and stack.

    Question 2

    What language was used to create the website?

    Answer: PHP

    Again I used Wappalyzer to figure out the answer to the question, being PHP. This question can also be answered by navigating to the uploads tab and looking at the address to see that it has a .php extension at the end of it.

    Question 3

    What version of this language is used?

    Answer: 5.5.9

    Lastly, this question was also answered with the information provided by Wappalyzer however, you could intercept the requests via Burp to find this information out also.

    Task 4

    Question 1

    What is the admin username?

    Answer: admin

    You must login to the admin portal to answer this question and through a lucky guess I was able to login as the user admin, this was an example of weak credentials.

    Question 2

    What is the admin password?

    Answer: admin

    In the same admin portal as above I entered admin as the password as a guess, which did work. The creator was definitely showing how weak credentials can leave a system open unintentionally.

    Question 3

    What is the name of the cookie that can be manipulated?

    Answer: session

    To figure out the cookie that can be manipulated you have to login to the admin portal and route the request through Burp, so that you can see the manipulable cookie being session.

    Question 4

    What is the username of a logged on user?

    Answer: bryce

    In order to answer this question you must first create an account on the site, it doesn't matter what you input because there is no validation system in place.

    Next, navigate to the recents tab and you'll see a listing of different pictures that you can click on. After clicking on the second one you'll be presented with the user that is currently logged onto the site to answer the question.

    Question 5

    What is the corresponding password to the username?

    Answer: bryce

    Funny enough, just like the admin portal having the same username and password, so does the user bryce.

    Task 5

    I used the first basic XSS test provided by the link on the machine and was able to get a response back denoted that the search bar is vulnerable to an XSS vulnerability( ).

    Testing for an XSS on the guestbook can be validated by inputting the payload via the comments section as that was triggers the actual vuln. This is a persistent(stored) XSS because it will trigger every time you go to the comments tab.

    The flash form being mentioned is on the home page right below the File Upload button. All I did is use the same basic XSS test to validate that this was a working vuln on the system.

    Question 1

    Using my current sample login I browse to the recent's tab and check the pictures looking at the URL. From there you see that the numbers go up by increments of 1, so go into the number 10 you're able to see another user's pictures, like with Bryce before.

    Question 2

    First, you need to click on the Upload tab then in the URL enter /../ then you can press enter.

    Which will bring you to a directory listing of the pictures that you aren't supposed to see.

    Question 3

    It doesn't matter which picture is added to your cart because they all work the same way. Once a picture is added to your cart click on the Continue to Confirmation button at the bottom

    Which will present you with a link to a high-quality resolution of the picture, which you shouldn't have been able to get yet, especially without a purchase.

    Question 4

    To get an item for free, first navigate to /calendar.php then click on the "What about tomorrow" button to be presented with a coupon like such.

    Once you have that coupon, place an image in your shopping cart. Then to get the item for free enter that coupon as many times at it takes for the item to become free. This works because the system doesn't prohibit the re-usage of the same coupon.
