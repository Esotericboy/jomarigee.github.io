    Machine Information

    Startup is a TryHackMe machine requiring you to "abuse traditional vulnerabilities via untraditional means".

    You can Join this TryHackMe Machine Here

    Information Gathering

    The target IP address is provided when the machine is deployed.

    Target: 10.10.124.127

    Scanning/Enumeration

    First I started off with a simple Nmap Scan to see what this machine was working with. I see that anonymous login is allowed on the FTP Server and there is the standard website along with an ssh port open. I'll go ahead and check out the website after my gobuster scan completes. 

    ➜ nmap -A 10.10.124.127

    Looking at the scan we can see the following ports and services are open:

    • ● Port 21/tcp - FTP - (vsftpd 3.03)
    • ● Port 22/tcp - SSH - (OpenSSH 7.2p2)
    • ● Port 80/tcp - HTTP - (httpd 2.4.18)

    Next thing I do is a gobuster scan on the IP range. Which pops back only two directories. The /files directory appears To be interesting, but I'll get to it later. 

    ➜ gobuster dir -u http://10.10.250.187 -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -n -e -x php,html,txt -t 30

    Website

    I visited the main website and was greeted with a very basic message saying that the dev team is still working on this site. Nothing here was helpful.

    Then I went to inspect element to see if there was a clue or comment hidden there, but nothing out of the ordinary. The only comment was a question of when would they update the site.

    FTP Server

    Now I decided to go and check out the FTP server that had come back from the Nmap scan. Looking into the FTP server there was only a text file and a jpg image. I downloaded both items to my machine to analyze.

    Looking over the downloaded text file notice.txt it just talks about Among Us jokes being on the server. The important thing it does give is a possible username maya.

    The next thing I do is look at the image that I downloaded. Which was as I assumed an Among Us meme. Nothing important here lol.

    Reverse Shell

    Now that I'd run through the FTP Server it was time to move onto the results I'd gotten from the gobuster scan. The first subdomain I went to was the /ftp one. While in there I navigated to the /files/ftp subdomain. Which this was weird because this is what was on the FTP server. This means if I can upload to the FTP server I have a place to get a reverse shell.

    I log back into the FTP server and with success am able to upload a php reverse shell onto the FTP server!

    ➜ send /home/kali/TryHackMe/php-reveres-shell/reverse.php shell.php

    Navigating back to the /files/ftp subdomain I see the shell!! So with my netcat listener already setup I click on the file to get the reverse shell.

    First Flag

    Now that I was in the system all it took to get the first flag or question was a cat command of the recipe.txt file.

    1. What is the secret spicy soup recipe?

    Answer: l***

    Wireshark

    After doing some more looking around the filesystem I found a weird folder called incidents with a Wireshark file in it. So I set a Python Web Server up and downloaded it to my machine to analyze.

    Using Wireshark and following the TCP Stream I found out that it was recording the session of another user, however I don't know who just yet.

    Looking deeper into the file I was able to found the SSH password for the user lennie. However, this could have been done easier had I used the command strings suspicious.pcapng, which had I known before this box would have made my life easier.

    User Flag

    With the credentials that I just got for Lennie. I login to the SSH service on the box with success!

    The user flag was easy to get because it just required a simple list command.

    2. What are the contents of user.txt?

    Answer: T***********************************

    Root Flag

    Using sudo-l I see that the folder scripts contains a file to gain root access. Going into this directory there is a file called planner.sh which is a cron job that can make me root. Using PayloadsAllTheThings I use the bash one liner via nano for root access.

    Once inside nano I entered my IP address and port that I was using for the netcat listener I had setup, then executed the command.

    Then once planner.sh was setup correctly I executed it. I had to give it a few seconds for the root shell because its a scheduled cron job, so root wouldn't be immediate.

    Now wonderful I'm root!!! All it takes to get the root flag is just a simple listing command and then boom the box is done!!

    3. What are the contents of root.txt?

    Answer: T***********************************

    Finished

    This was a fun box to do! It was my first time messing around with cron jobs, so it was cool to understand how they worked.☺
