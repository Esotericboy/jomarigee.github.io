    Machine Information

    Mr.Robot is an intermediate CTF that challenges you to find three flags or as they are referred to on this machine keys.

    You can Join this TryHackMe Machine Here

    Information Gathering

    The target IP address is provided when the machine is deployed.

    Target: 10.10.90.76

    Scanning/Enumeration

    Running an nmap scan on the target I get back port 80 and 443 open, but strangely port 22 is closed. 

    ➜ nmap -A 10.10.90.76

    Looking at the scan we can see the following ports and services:

    • ● Port 22/tcp - SSH
    • ● Port 80/tcp - HTTP
    • ● Port 443/tcp - HTTP

    My gobuster scan bounced back a ton of directories, more then I've ever seen on any box before. So in order not fall too hard down the rabbit hole I'll go through the directories in order to see what I find. 

    ➜ gobuster dir -u http://10.10.90.76 -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -n -e -x php,html,txt -t 30

    First Key

    I know that this is a web application, so I figured I would check for a robots.txt page. Low and behold that is where the first key was sitting, which is cool.

    1. What is key 1?

    Answer: 0*******************************

    Along with finding the flag, I also found a directory listing which held an important file. This file will come in handy when I'm trying to brute force a login.

    Pwn Login Portal

    I went to the login sub-directory at 10.10.90.76/wp-login.php and was presented with a login page for a Wordpress application, nice. I used the wordlist found from the robots page to brute force the page via Hydra. 

    ➜ hydra -L fsocity.dic -p admin 10.10.90.76/admin http-post-form "/wp-login.php:log=^USER^&pwd=^PWD^:Invalid username." -t 30 -I -o hydra.txt

    It took some time, but Hydra gave me a username and what I thought was a working password to login with.

    However the password admin was not correct. So to save time I used the tool WPScan since its a wordpress application and this time it was much faster then before.

    ➜ wpscan --url 10.10.90.76/wp-login.php -t 50 -U elliot -P fsocity.dic

    Reverse Shell

    Now that I'm successfully logged into the WordPress application its time to walk around it and make myself at home.

    Visiting the other users tab on the website I was told about Elliot's therapist Krista. In her biography portion it asked the questions, "another key"? However even after completing this box and looking back through it I couldn't find the supposed key mentioned here.

    In the edit themes archive section I found a place to upload my reverse shell. Once the shell was uploaded I went to where it was uploaded so that the shell could be activated.

    ➜ http://10.10.90.76/wp-content/themes/twentyfifteen/archive.php

    Second Key

    With my shell first thing I did was upgrade it to a fully privileged shell. Navigating around I see the second key, but I'm unable to access it yet due to my current permissions. However, the md5 password I need to login as the user robot is there, so I use Crackstation to crack it.

    Using the password I login as the user robot and then cat out the second key.

    2. What is key 2?

    Third Key

    For the third key I had to figure out which program had a misconfigured SUID of 4000, so that I could escalate my privileges and become root.

    ➜ find / -perm /4000 -print 2>/dev/null

    After some researching and trying things out I was able to find something on GTFOBins and was able to get the final key to finish the box.
