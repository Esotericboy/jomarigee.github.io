    Information Gathering

    The target IP address is provided when the machine is deployed.

    Target: 10.10.10.3

    Scanning/Enumeration

    I start off with an nmap scan to see what I'm working with and get back a handful of ports, but no website. However, I do have a great deal of information on this machine and have my eyes looking into the Samba share service that is present. 

    ➜ nmap -Pn -A -p- 10.10.10.3 -o nmap.txt

    Looking at the scan we can see the following ports and services are open:

    • ● Port 21/tcp - FTP - (vsftpd 2.3.4)
    • ● Port 22/tcp - SSH - (OpenSSH 4.6p1)
    • ● Port 139/tcp - NetBios-SSN - (smbd 3.X - 4.X)
    • ● Port 445/tcp - NetBios-SSN - (smbd 3.0.20)
    • ● Port 3632/tcp - Distccd - (Ubuntu 4.24)

    Passing the open ports to a vulnerable nmap scan I was informed about about a wonderful CVE that port 3632 on this machine was vulnerable to.

    ➜ nmap -Pn -p21,22,139,445,3632 --script vuln 10.10.10.3 -o nmapvuln.txt

    FTP Enumeration

    However, before I go through with executing that CVE I want to check out the FTP Server because Anonymous login is allowed and something good could be on there.

    Much to my surprise I didn't find anything in the server, even when I browsed it via a web interface.

    CVE Exploitation

    Going back to the Samba service that is in use. I use searchsploit to find a CVE for Samba 3.0.20 which after reading over I decide to pivot to Metasploit for exploitation.

    ➜ searchsploit samba 3.0.20

    I searched up the correct module in Metasploit and then entered use 14, so that I could use it.

    After setting the proper LHOST and RHOSTS I checked my options again and was able to exploit the machine.

    User Flag

    Once the machine has been exploited I wanted to drop into a shell. After doing that I ran a simple whoami command and found out that I'm already root!!

    Then I simply just navigate into the user makis' directory and cat out the user flag.

    Root Flag

    Now since I'm already the root user of this box, all I need to do is navigate to the root directory and cat out the file.

    Finished
