    • Machine Information

    SimpleCTF is a beginner level CTF that requires you to use a CVE, so that you can find the user and the root flag for the box.

    You can Join this TryHackMe Machine Here

    Information Gathering

    The target IP address is provided when the machine is deployed.

    Target: 10.10.158.221

    Scanning/Enumeration

    I run an nmap scan to see what kind of machine I'm dealing with and get the following three ports are open.

    ➜ nmap -sC -sV -A 10.10.158.221

    Looking at the scan we can see the following ports and services are open:

    • ● Port 21/tcp - FTP - (vsftpd 3.03)
    • ● Port 80/tcp - HTTP - (httpd 2.4.18)
    • ● Port 2222/tcp - SSH - (sshd 7.2p2)

    1. How many services are running under port 1000?

    Answer: 2

    2. What is running on the higher port?

    Answer: ssh

    While my nmap scan was going I ran a gobuster scan to enumerate for more sub-directories and got an interesting one pop back called simple

    ➜ gobuster dir -u http://10.10.158.221 -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -n -e -x php,html,txt -t 30

    CVE Exploit

    The simple domain is a page for a CMS called CMS Made Simple, which is whats needed to figure out what CVE will be used for this machine.

    Searching on Exploit-DB I found out that the CMS was vulnerable to CVE-2019-9053 which is an SQL Injection.

    3. What's the CVE you're using against the application?

    Answer: CVE-2019-9053

    4. To what kind of vulnerability is the application vulnerable?

    Answer: sqli

    After downloading the CVE and using python2 to get it to work I was given a username of mitch along with a hashed password

    5. What's the password?

    Answer: secret

    SSH

    Once John The Ripper cracked the password I was able to login as Mitch into ssh on port 2222 because ssh wasn't running on port 22 for this box.

    6. Where can you login with the details obtained?

    Answer: ssh

    User Flag

    The first flag was simply sitting in Mitch's directory which was nice because it just required a simple ls command.

    7. What's the user flag?

    Answer: G*****************

    Then to answer the other users present question I just bounced back a directory and found the username.

    8. Is there any other user in the home directory? What's its name?

    Answer: s******

    Root Flag

    The last thing I needed to was get the root flag so I ran sudo -l so that I knew what it took to become root. Which is awesome because to become root you don't need a password!!

    9. What can you leverage to spawn a privileged shell?

    Answer: v**

    Thanks to this article I was able to become root on the machine and get the flag to complete it.

    10. What's the root flag?

    Answer: W**********************

    Finished

    This machine was like the title suggested, pretty simple and straight forward. Really fun because I felt good for once.☺
