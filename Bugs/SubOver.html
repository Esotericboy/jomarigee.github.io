    Introduction

    SubOver is a tool that can be used for testing subdomain takeovers however, it is no longer receiving updates anymore. Even though this project is no longer being provided updates the version it was left in is a great state. Usable and can defiantly be improved upon down the road for sure.

    Installation

    First when installing you must have Golang then just enter the command below and you'll be able to use the tool.

    ➜ go get github.com/Ice3man543/SubOver

    You have two choices on how you can run this tool. You can either run it from its location in you go folder such like the picture below.

    Or you can add the go program to your PATH directory, so that it can be run anywhere. For the sack of clarity in this article I'll be running it from where I've put in my PATH folder. You can use either the cp or mv command to move it, doesn't matter as long as it's in your PATH folder.

    Usage

    As far as usage goes, this tool isn't like FFUF or Amass where they have more than one function inside of them. This tool is a pretty straightforward subdomain takeover tool. You basically just point shoot such as with the picture below.

    This tool is so straightforward there aren't really tricks to using it, so the command below will pretty much do you great. Of course, increase the amount of thread to your liking depending on your CPU.

    Errors

    When first using this tool you're most definitely going into the same error I and many other users ran into, being open providers.json: no such file or directory

    This error can be solved in two different ways. You can add the providers.json file to your PATH, so that this error doesn't pop up when using SubOver globally

    ➜ sudo cp ~/go/src/github.com/Ice3man543/SubOver/providers.json

    I didn't do this because I like to keep my PATH folder as clear as possible, so to get around this I just run the command from the SubOver folder. All I have to do is specify where my host file is, which the one in my demo is located in my /home/kali/takeovers.json directory or ~/takeover.json

    How I Use It

    I use this tool when after running aquatone-takeover as a second check to see if a subdomain is still vulnerable to a takeover before I go out and check the potentially vulnerable domain manually.

    It also serves as good to use when you have a list of domains that you want to check because Aquatone didn't check them for some. Unlike Aquatone SubOver doesn't require you to jump through hoops in order to use it.

    Finished

    All in all this tool is great to add to your workflow when checking a target for takeovers and works great when used in conjunction with tools such as Aquatone and SubJack.☺
